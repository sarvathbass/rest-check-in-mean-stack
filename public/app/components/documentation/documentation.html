Documentation

Introduction

Here you can find a list of core libraries, design specifications and coding standards that we use in Mara.

Google's Material Design

All libraries and custom made components are following Google's Material Design Specifications.

Materialize

Mara is built upon the new Materialize css framework.

HTML/jQuery & AngularJS

When creating your project you can choose to use the standard HTML/jQuery version and the AngularJS version.

If you don't know what AngularJS is or don't know how to use it, we strongly recommend checking out AngularJS.

Out of the box version

In addition to the standard version of the theme that requires Node.js and Grunt in order to be installed,
we also bundle an installed version that can just be extracted and used.
If you decide to use the installed version, you don't need to follow the steps from the Prerequisites and the Installation sections below.

Prerequisites

In order to start using Mara you will need some tools for installing 3rd party components and libraries

Node.js

Node.js® is a JavaScript runtime built on Chrome's V8 JavaScript engine.

Grunt

Grunt and Grunt plugins are installed and managed via npm, the Node.js package manager.

Installation
In this section we will be installing the necessary tool and 3rd party components and libraries
Installing Prerequisites
  1. Download and install the latest Node.js from its web site.

  2. Run this command with a CLI client to install Grunt:

                 
                 npm install -g grunt-cli
Installing Mara

Navigate to the theme's folder and follow the following steps to complete installation

  1. Run this command to install 3rd party components and libraries

                 
                 npm install

  2. Now run this custom created command to have Grunt compile all the CSS and jQuery assets

                 
                 grunt deploy

And that's it, your are now ready to use Mara!

Creating your own pages

To start using Mara you first have to include the compiled CSS file:

Code Sample
                                                .
<link type="text/css" rel="stylesheet" href="css/mara.min.css"  media="screen,projection"/>

And 3 javascript files ( jQuery, Materialize framework and the custom compiled javascript files ):

Code Sample
                                                .
<script type="text/javascript" src="//code.jquery.com/jquery-2.1.1.min.js"></script>
<script type="text/javascript" src="js/materialize.js"></script>
<script type="text/javascript" src="js/mara.min.js"></script>

We also created a custom Grunt command that can be used in the development process. It watches inside the /assets/sass/ and /assets/js/scripts/ folders for any changes and when it detects one, it starts re-compiling the files.

To start using it just type:

             
                 grunt w
3rd Party Components

In this section we will be presenting some of the components and plugins used.

AmCharts

amCharts s an advanced charting library that will suit any data visualization need.
Their charting solution include Column, Bar, Line, Area, Step, Step without risers, Smoothed line, Candlestick, OHLC, Pie/Donut, Radar/ Polar, XY/Scatter/Bubble, Bullet, Funnel/Pyramid charts as well as Gauges.
It perfectly segregates itself into 3 independent charts — JavaScript Charts, Maps Charts (amMaps), and Stock charts.
amCharts uses SVG to render charts that work in modern browsers only. These charts may not work properly below Internet Explorer 9.
Checkout this fantastic collection of cool demos created using amCharts.
And also these cool tips (from where these examples were selected) to help you customize your amCharts charts.
The free version of amChart will leave a link to its website on the top of every chart. The commercial licenses are bit more expensive than most of the paid libraries in the market.

C3.js

C3.js is a D3-based reusable chart library.
C3 makes it easy to generate D3-based charts by wrapping the code required to construct the entire chart.
C3 gives some classes to each element when generating, so you can define a custom style by the class and it's possible to extend the structure directly by D3.
C3 provides a variety of APIs and callbacks to access the state of the chart. By using them, you can update the chart even after it's rendered.
Because of the dependence on D3, C3 supports only modern browsers D3 supports. Please see the description in D3.

Flot

Flot is a pure JavaScript plotting library for jQuery, with a focus on simple usage, attractive looks and interactive features.
It is also one of the oldest and popular charting libraries.
Flot supports lines, points, filled areas, bars and any combinations of these.
Flot is completely free to use and commercial support is provided on special request to the creator.
Works with Internet Explorer 6+, Chrome, Firefox 2+, Safari 3+ and Opera 9.5+
Please see their examples.

Highcharts

Highcharts is a charting library written in pure JavaScript, offering an easy way of adding interactive charts to your web site or web application.
Designed from ground up with mobile browsers in mind, everything from multitouch zooming to touch-friendly tooltips responds great on mobile platforms.
In modern browsers graphs are rendered in SVG, with VML support for legacy browsers (e.g. Internet Explorer 6).
Highcharts’ fast, mature, and modular codebase is also the foundation of its two sister products, Highstock JS and Highmaps JS.
While Highchats is free for personal use, you need to purchase a license for commercial usage.

Morris js

morris.js is a pretty and powerful Charts Plugin with jQuery and Raphaël JS Library to make drawing simple charts easy.
With Morris.js, you can create a wide variety of charts like line & area charts, bar charts and donut charts to fit you needs.
It also works fine on mobile device such as iOS and android
It was created by Olly Smith.

nvD3.js

nvD3.js is a D3-based reusable chart library.
This project is an attempt to build re-usable charts and chart components for d3.js without taking away the power that d3.js gives you.
This is a very young collection of components, with the goal of keeping these components very customizeable, staying away from your standard cookie cutter solutions.
Checkout their examples gallery for how to code your first chart.
Their examples should provide you with enough code to start making beautiful charts.
Because of the dependence on D3, C3 supports only modern browsers D3 supports. Please see the description in D3.

jQuery Sparkline

jQuery Sparkline generates sparklines (small inline charts) directly in the browser using data supplied either inline in the HTML, or via javascript.
The plugin is compatible with most modern browsers and has been tested with Firefox 2+, Safari 3+, Opera 9, Google Chrome and Internet Explorer 6, 7, 8, 9 & 10 as well as iOS and Android.
Each example displayed below takes just 1 line of HTML or javascript to generate.
The plugin was written by Gareth Watts for Splunk Inc and released under the New BSD License.

FAQ

This section provides answers for frequently asked questions

How to remove theme settings button?

The easiest way is to add the following code snippet in your HTML pages: 

<script>window.disableThemeSettings = true;</script>

Another, more permanent way, is to comment out the code from the '\assets\js\scripts\_theme-settings.js' file that creates the theme settings button: 

$('body').append(
    $('<div/>').load('/templates/themeSettings.html', function(){
        ThemeSettings.init();
    })
);

And then run the 'grunt deploy' or the 'grunt w' command to repack the theme's JS components into the 'mara.min.js' file.
